# Demo Scenario

## 3分デモ

### 1. 起動

```powershell
openclaw-companion
```

Companionが最初に声をかけます。

```text
Claw いるよ。今日は横で一緒に進めよう。まず一手だけ切ろう。
```

### 2. タスクを依頼

入力欄に入れます。

```text
READMEの改善点を洗い出して
```

ログに開始、分析、提案、完了が流れます。

```text
Claw 了解。`READMEの改善点を洗い出して` を見に行く。進み具合はここに流すね。
Claw タスクを読んで、作業単位に分けてる。
Claw 関連しそうなファイルとコマンドを想定してる。
Claw MVPではここでOpenClawに処理を渡す設計。
Claw 一区切り。今の一歩、ちゃんと前に進んでる。
```

### 3. 実コマンド

```text
!dir
```

PowerShellコマンドの結果が会話ログに混ざります。実行係でありながら、結果に対して人っぽく締めます。

### 4. 集中フィードバック

しばらくキーボード入力を続けると:

```text
Claw いい感じ、そのまま行こう。
```

ウィンドウ切替が増える、または無操作が続くと:

```text
Claw ……さっきから窓が忙しいね。主役のターミナルに戻る？
Claw 5分近く止まってる。詰まった場所だけ言葉にしてみる？
```

## 見せたい価値

- CLIツールなのに、一人で作業している感じが少し薄れる
- 進捗ログが冷たい通知ではなく、共同作業者の声に見える
- 脱線や停止を責めずに、作業へ戻るきっかけを作る
- 既存のOpenClawに薄く重ねられる
